## Included Workflows

- Nightly build: `.github/workflows/nightly-android-build.yml`
- Promote to release: `.github/workflows/promote-to-release.yml`

## What each pipeline does

1. `Nightly Android Build`
   - Runs every night (and manually if needed)
   - Builds release APK + Play Store AAB
   - Publishes artifacts (`apk`, `aab`, `SHA256SUMS.txt`) for 14 days

2. `Promote Nightly to Release`
   - Manual workflow (`workflow_dispatch`)
   - Takes a nightly `run_id`
   - Creates GitHub release/tag `vX.Y.Z` using nightly artifacts

## Production Setup (one-time)

### 1) Repository settings

- Enable GitHub Actions for this repository.
- In `Settings -> Actions -> General`, allow read/write permissions for `GITHUB_TOKEN`.
- (Recommended) Create environment `production` in `Settings -> Environments` and add reviewers for release approval.

### 2) Android signing secrets

The nightly workflow supports production signing if these repository secrets are configured:

- `FREEKIOSK_UPLOAD_STORE_FILE_B64`
- `FREEKIOSK_UPLOAD_STORE_PASSWORD`
- `FREEKIOSK_UPLOAD_KEY_ALIAS`
- `FREEKIOSK_UPLOAD_KEY_PASSWORD`

Create Base64 keystore content locally:

```bash
base64 -w 0 release.keystore
```

Use the output as `FREEKIOSK_UPLOAD_STORE_FILE_B64`.

> [!IMPORTANT]
> If these secrets are missing, the build falls back to debug signing and must not be promoted to production.

## Nightly Operations

- Workflow: `Nightly Android Build`
- Triggered automatically by cron, or manually from Actions tab.
- Artifacts generated by the workflow:
  - `freekiosk-nightly-apk`
  - `freekiosk-nightly-aab`
  - `freekiosk-nightly-checksums`

## Promote to Release Operations

1. Open `Actions -> Nightly Android Build`.
2. Copy the successful `run_id` you want to promote.
3. Run `Promote Nightly to Release` with:
   - `version`: `X.Y.Z` (example `1.2.18`)
   - `nightly_run_id`: the run ID from step 2
   - `prerelease`: `true/false`
   - `release_notes`: optional extra notes

The workflow resolves the nightly commit SHA, creates tag `vX.Y.Z`, and publishes the GitHub release with APK/AAB/checksums.

## Recommended Production Policy

- Keep `Promote Nightly to Release` restricted to maintainers.
- Require approvals on `production` environment.
- Promote only nightly runs built from protected branch `main`.
- Verify generated checksums before external distribution.

## Troubleshooting

- `Artifact not found`: check `nightly_run_id` and verify run success.
- `Release already exists`: use a new version tag.
- `Invalid version`: expected `MAJOR.MINOR.PATCH` format (optional suffix allowed).
- Signing fallback detected: configure missing signing secrets and rebuild nightly.
